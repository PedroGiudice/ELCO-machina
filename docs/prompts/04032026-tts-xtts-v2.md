# Retomada: Benchmark XTTS v2 no Modal para TTS PT-BR

## Contexto rapido

Estamos adicionando TTS via Modal ao ELCO-machina. O STT (Whisper) ja funciona perfeitamente (`scripts/modal_whisper_bench.py`, T4, $0.001/transcricao). Para TTS, testamos F5-TTS exaustivamente e descartamos: o checkpoint PT-BR (`firstpixel/F5-TTS-pt-br`) gera gibberish, e o modelo base produz PT-BR reconhecivel mas com qualidade insuficiente para producao.

A decisao e migrar para **XTTS v2 (Coqui TTS)** -- unico modelo open-source com PT-BR como idioma de treinamento nativo, co-criado por Edresson Casanova (USP/NVIDIA). Suporta voice cloning zero-shot.

**Regra desta sessao:** so escrever/editar scripts .py de benchmark. Nenhuma alteracao no app.

## Arquivos principais

- `scripts/modal_f5tts_bench.py` -- benchmark F5-TTS (usar como base/referencia para XTTS v2)
- `scripts/modal_whisper_bench.py` -- benchmark Whisper (padrao ouro de script Modal)
- `modal_functions/tts_chatterbox.py` -- Chatterbox TTS reescrito (nao testado, manter como fallback)
- `docs/Refaudio.wav` -- 10s ref audio PT-BR para testes de voice cloning
- `docs/contexto/04032026-tts-f5tts-benchmark.md` -- contexto detalhado desta sessao
- `docs/F5-TTSPT-BR.txt` -- pesquisa sobre F5-TTS PT-BR (util para comparacao)

## Proximos passos (por prioridade)

### 1. Criar script benchmark XTTS v2 no Modal
**Onde:** `scripts/modal_xtts_bench.py` (novo)
**O que:** Script Modal seguindo padrao do Whisper -- modelo na imagem, L4 GPU, voice cloning com `docs/Refaudio.wav`
**Por que:** XTTS v2 e o unico modelo com PT-BR nativo + voice cloning zero-shot
**Verificar:** `modal run scripts/modal_xtts_bench.py --ref-audio docs/Refaudio.wav --text "Texto de teste em portugues."` seguido de transcricao com Whisper para validar qualidade

**Pontos tecnicos:**
- Pacote: `TTS` (Coqui TTS), modelo `tts_models/multilingual/multi-dataset/xtts_v2`
- XTTS v2 gera audio a 24kHz
- Ref audio ideal: 5-9 segundos, sem ruido
- Acentuacao obrigatoria no gen_text
- API: `tts.tts(text=..., speaker_wav=..., language="pt")`

### 2. Validar qualidade com Whisper
**Onde:** usar `scripts/modal_whisper_bench.py` existente
**O que:** transcrever output do XTTS v2 e comparar com input
**Por que:** metodo objetivo de validacao (ja usado em todos os testes F5-TTS)
**Verificar:** `modal run scripts/modal_whisper_bench.py --audio-path /tmp/xtts_output.wav --language pt`

### 3. Se XTTS v2 funcionar, comparar metricas com F5-TTS
**O que:** documentar latencia, custo, qualidade lado a lado
**Referencia F5-TTS base:** 2.0s warm, ~10s audio, $0.45/1000, qualidade insuficiente
